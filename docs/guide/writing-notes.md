# 笔记编写规范

## 文件命名

文件和目录统一使用小写英文与连字符，例如：

```text
notes/python/functions-modules.md
```

清晰的文件名可以产生稳定、容易分享的页面地址。

## 推荐结构

一篇学习笔记通常包含以下部分：

1. 这个知识点解决什么问题。
2. 最小可运行示例。
3. 常见错误或容易混淆之处。
4. 实际应用场景。
5. 后续可继续阅读的主题。

## Markdown 示例

````markdown
# 页面标题

一句话说明本文目标。

## 核心概念

正文内容。

```python
print('可运行示例')
```

::: warning 注意
这里写容易踩坑的地方。
:::
````

## 内容检查

- 标题是否准确描述内容。
- 内部链接是否能正常打开。
- 代码是否能够独立运行。
- 是否删除了无用截图和重复段落。
- 专有名词的写法是否前后一致。
